Der Gorilla-Test ist effektiv 32 Tests auf einmal: Dafür werden von 26 32-Bit Ganzzahlen die Bits neu angeordnet, sodass daraus 32 26-Bit Ganzzahlen werden. Die erste Zahl verwendet immer das höchstwertige Bit (31), die zweite Zahl immer das zweithöchste Bit (30) und so weiter, bis schließlich die 32. Zahl immer nur das niedrigstwertige Bit (0) verwendet.

Es werden immer 134217728 solcher Zahlen-Gruppen erzeugt, für jede individuelle Gruppe einzeln mit überlappenden Bits. Jede so erzeugte Zahl wird innerhalb ihrer Gruppe als "erzeugt" markiert und am Ende wird innerhalb jeder Gruppe gezählt, wie viele Zahlen nicht erzeugt worden sind. Die Anzahl der nicht erzeugten Zahlen sollte einer Normalverteilung folgen mit einem Mittelwert von 24687971 und einer Standardabweichung on 4170.
